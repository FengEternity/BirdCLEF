### Exp #4: Stage 2 域适应训练（Train V11 → NaN 失败）

**日期**: 2026-03-26  
**结果**: ❌ Ep 01 loss=NaN, AUC=0.4964（随机水平）

**模型配置变更（相对基线）**:
- GeM Pooling (p=3.0, 可学习) 替代 global_pool='avg'
- 分类头升级：FC(1280→512) + ReLU + Dropout(0.2) + FC(512→234)
- Dropout 堆叠：backbone 后 0.3 + 分类头内 0.2

**损失函数变更**:
- AsymmetricLoss 替代 BCEWithLogitsLoss
  - gamma_neg=4.0, gamma_pos=1.0, clip=0.05
  - Per-class pos_weight（纲级加权 × 频率逆权重）

**数据管线变更**:
- 声景弱标签混入 30%、背景噪声增强 40%

**NaN 根因分析**:
1. **ASL eps 溢出**（主因）：`eps=1e-8` 在 AMP float16 中舍入为 0 → `log(0) = -inf` → NaN
2. **pos_weight 极端值**（次因）：零/稀有物种权重可达数百，放大 inf/NaN
3. **GradScaler 放大**：65536x scale 使大 loss 溢出 float16

### Exp #5: Stage 2 修复 + 10s 训练对齐（Train V12，训练中）

**日期**: 2026-03-26

**相对 V11 的变更**:

| 变更 | V11 | V12 | 原因 |
|------|-----|-----|------|
| ASL 精度 | AMP float16 | `@custom_fwd(cast_inputs=float32)` | 防止 log(0) NaN |
| ASL eps | 1e-8 | **1e-4** | float16 安全 |
| pos_weight | 无上界 | **clip(0.1, 10.0)** | 防止极端权重 |
| CLIP_DURATION | 5s | **10s** | 训练-推理对齐（2024 冠军方案） |
| BATCH_SIZE | 64 | **32** | 10s mel 2x 宽，对应减半 |
| TMASK_W | 50 | **100** | 时间掩码随 mel 宽度等比增大 |
| 训练偏移 | 0-25s | **0-20s** | 适配 10s 截取 |
| 噪声偏移 | 0-55s | **0-50s** | 适配 10s 截取 |

**实际结果**:
- ✓ NaN 问题消除（loss 正常下降）
- ✗ AUC 停滞在 ~0.53，3 个 epoch 无改善
- **根因**：ASL+pos_weight 使 loss 与 AUC 完全不对齐——模型优化 loss 但不改善排序

```
Ep 01: loss=1.359  AUC=0.531  (loss 正常, AUC 几乎随机)
Ep 02: loss=0.803  AUC=0.526  (loss -41%, AUC -0.005)
Ep 03: loss=0.660  AUC=0.528  (loss -18%, AUC +0.002)
```

**教训**：同时改 6 个变量无法定位问题。必须转为单变量消融实验。

## 关键学习

### L1: 推理优化的理论天花板（Stage 1 深度分析）

**问题**：Stage 1 预期 +0.05，实际仅 +0.01。为什么？

**根因**：ROC-AUC 仅取决于预测的**排序**（ranking），不受绝对值影响。

当前模型在声景上的排序能力（AUC 0.683）是由**特征表示质量**决定的。推理优化无法改变特征质量——它只能通过更好的聚合微调排序。

**逐项分析**：

| 优化 | 预期 | 实际 | 失败原因 |
|------|------|------|---------|
| **10s 上下文** | +0.03 | ~+0.005 | 2024 冠军训练+推理都用 10s；我们仅改推理端。EfficientNet 的 global avg pool 将信号稀释 2x（叫声从占 10% → 5% 时间帧），模型从未学习如何利用额外上下文。 |
| **时间平滑** | +0.01 | ~+0.003 | 正样本预测均值仅 0.067，负样本 0.003。相邻窗口预测太低，平滑反而**降低**峰值预测（从 0.067 降到 0.048）。只有持续叫声受益，短暂叫声反受害。 |
| **后处理修复** | +0.01 | ~+0.002 | sonotype 除法只影响 10/234 种（4%）；共现阈值 0.05 在声景上仍很少触发（大多数预测 < 0.01）。 |

**数学边界**：
- 声景 AUC 0.683 = 模型对 68.3% 的正/负样本对排序正确
- 最优推理聚合的上界估计：+0.01~0.02
- 实际 +0.01 已接近理论上界

**核心洞察**：
- 温度缩放对 AUC **零影响**（单调变换不改变排序）
- 517063 物种的正样本预测（8.9e-06）**低于**负样本（1.9e-05）→ 模型对该物种的排序**完全反向**
- **任何推理优化都无法修复特征空间中迷失方向的模型**

**结论**：推理优化天花板 = 声景 AUC 对应的 LB 上限。要突破必须通过 Stage 2 训练改变特征表示。

### L2: 10s 上下文的训练-推理一致性原则

模型只能有效利用训练时见过的输入模式。给训练在 5s 上的模型喂 10s 输入：
- 梅尔图宽度 2x → 特征图尺寸不同 → 全局池化行为改变
- 等效于把"近距离照片"换成"远距离全景"——模型没学过如何解读全景
- **必须训练和推理保持一致**才能获得完整收益

### L3: 低概率区域的平滑陷阱

当预测值普遍极低时（< 0.01），时间平滑的**方向**取决于邻居：
- 如果邻居也低 → 平滑无效甚至有害（削峰）
- 只有邻居也高时才有正面效果
- 这要求模型在声景上有基本的检测能力——我们没有
