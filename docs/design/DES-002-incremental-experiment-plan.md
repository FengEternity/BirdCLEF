# DES-002: 增量实验计划（V2 修订版）

> **核心原则**：每次实验只改变一个变量，精确衡量每个改动的独立贡献。
>
> **基准线**：~~V8 基线（CV AUC 0.9776, LB 0.779）~~ → **V14（A1, CV 0.9721, LB 0.830）**
>
> **V2 修订依据**：ANAL-005 诊断发现瓶颈从"域偏移"转为"声景多样性"，伪标签提前到第二优先级。

## 教训总结

| 版本 | 改动数量 | 结果 | 教训 |
|------|---------|------|------|
| V11 | 6 个同时改动 | loss=NaN | ASL eps 溢出 + 太多变量无法定位 |
| V12 | 修复 NaN + 再加 10s | AUC 停滞 0.53 | ASL+pos_weight 导致 loss 与 AUC 不对齐 |
| **V14 (A1)** | **基线 + 声景 30%** | **LB 0.830 (+0.041)** | **单变量消融有效，域适应收益最大** |

**A1 核心洞察**：训练声景 AUC 0.995 vs LB 0.830 → 0.165 的"声景间泛化差距"。瓶颈不再是域偏移，而是声景站点多样性不足。

## 实验执行规则

### 1. 单变量原则

- 每次实验**严格只改变一个配置项**
- 如果一个改动需要伴随其他修改（如 10s 需要 batch_size 减半），算作同一变量

### 2. 决策规则

- **LB 提升 ≥ 0.005**：采纳改动，进入下一实验
- **LB 变化 < 0.005**：无效改动，跳过或标记低优先级
- **LB 下降 > 0.01**：回退改动

### 3. 训练-推理一致性

- 模型架构必须在训练和推理 notebook 中**完全一致**
- 每次改动模型架构时，**同时更新**两个 notebook

### 4. 文档更新

- 每次实验结果出来后，更新 EXP-001 实验日志
- 记录：版本号、改动摘要、CV AUC、LB Score、分析
