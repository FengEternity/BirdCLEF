

# DES-002: 增量实验计划

> **核心原则**：每次实验只改变一个变量，精确衡量每个改动的独立贡献。
>
> **基准线**：V8 基线（CV AUC 0.9776, LB 0.779）

## 教训总结（V11-V12 失败复盘）


| 版本  | 改动数量            | 结果          | 教训                               |
| --- | --------------- | ----------- | -------------------------------- |
| V11 | 6 个同时改动         | loss=NaN    | ASL eps 溢出 + 太多变量无法定位            |
| V12 | 修复 NaN + 再加 10s | AUC 停滞 0.53 | ASL+pos_weight 导致 loss 与 AUC 不对齐 |


**结论**：必须逐步添加改动，每次只变一个。

## 实验执行规则

### 1. 单变量原则

- 每次实验**严格只改变一个配置项**
- 如果一个改动需要伴随其他修改（如 10s 需要 batch_size 减半），这些"必要伴随修改"算作同一个变量

### 2. 决策规则

- **LB 提升 ≥ 0.005**：采纳这个改动，进入下一个实验
- **LB 变化 < 0.005**：视为无效改动，跳过或标记为低优先级
- **LB 下降 > 0.01**：回退这个改动

### 3. 训练-推理一致性

- 模型架构必须在训练和推理 notebook 中**完全一致**
- 每次改动模型架构时，**同时更新**两个 notebook

### 4. 文档更新

- 每次实验结果出来后，更新 EXP-001 实验日志
- 记录：版本号、改动摘要、CV AUC、LB Score、分析
