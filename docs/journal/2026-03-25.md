# 2026-03-25 工作日志

## Kaggle 训练调试（核心任务）

### 问题 1：P100 GPU CUDA 不兼容
- Kaggle 分配 P100（SM 6.0），PyTorch 2.10+cu128 不支持
- 报错：`CUDA error: no kernel image is available for execution on the device`
- 解决：添加 GPU 能力自动检测，SM < 7 时安装 torch 2.5.1+cu124

### 问题 2：torchaudio.info() API 不存在（**根本 Bug**）
- `torchaudio 2.10` 移除了 `torchaudio.info()` 函数
- `load_audio_fast` 中 `torchaudio.info(path)` 抛出 `AttributeError`
- `except` 分支默默返回零向量，所有音频变成静音
- **导致模型训练 12 epoch AUC 始终 = 0.50（随机水平）**
- 解决：移除 `torchaudio.info()`，改为 `torchaudio.load()` 全量加载 → 重采样 → 截取

### 问题 3：DataParallel 对小模型无效
- B0（4.8M 参数）GPU 计算很快，CPU 数据加载是瓶颈
- 双卡 DataParallel 比单卡更慢（6.36s/it vs 4.28s/it）
- 通信开销 > 并行收益

### 修复后结果
- V7b：简化模型 + 修复 torchaudio 加载
- **Epoch 1 AUC = 0.8124** → **最终 Best AUC = 0.9755**（epoch 9, 191/234 物种有效）
- 训练曲线：0.81 → 0.92 → 0.95 → 0.96 → 0.97 → 0.97 → 0.97 → 0.97 → **0.976** → 0.975
- 正负样本分离：epoch 1 的 9.3x → epoch 9 的 342x
- 总训练时间：~82 分钟（单卡 T4, 10 epoch）
- epoch 10 AUC 微降（0.975 < 0.976），表明 10 epoch 已充分

## 经验总结

1. **异常处理掩盖 Bug**：`except Exception` 返回默认值，让数据加载失败完全隐形。诊断打印至关重要。
2. **API 版本兼容**：torchaudio 2.x 移除了多个旧 API，不能假设接口存在。
3. **小模型不需要多 GPU**：EfficientNet-B0 级别的模型，单卡 T4 足够，DataParallel 通信开销反而拖累。
4. **先简化再排查**：移除 class weights / focal loss / GeM 等复杂组件，用最简配置验证数据管道正确性。
