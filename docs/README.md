│
├── analyze/                  ← 分析文档（ANAL-XXX 编号）

## 子目录阅读卡片

### 📊 analyze/ — 分析文档

| 字段 | 说明 |
|------|------|
| **用途** | 竞赛调研、数据分析、技术方案对比、排行榜分析 |
| **编号规则** | `ANAL-XXX-简短描述.md`，从 001 递增 |
| **当前文档** | `ANAL-001-competition-analysis.md` — 竞赛深度分析 |
| **何时阅读** | 需要了解竞赛背景、技术路线、历年方案时 |
| **何时新建** | 做数据 EDA、对比模型方案、分析排行榜变化时 |

### 🏗️ design/ — 设计方案

| 字段 | 说明 |
|------|------|
| **用途** | 模型架构设计、数据增强策略、集成融合方案、训练流水线设计 |
| **编号规则** | `DES-XXX-简短描述.md`，从 001 递增 |
| **当前文档** | （暂无） |
| **何时阅读** | 实现模型或流水线前，确认设计方案 |
| **何时新建** | 确定新的模型架构、增强策略、集成方案时 |

### 🧪 experiment/ — 实验记录

| 字段 | 说明 |
|------|------|
| **用途** | 每轮训练的超参数、指标、对比结果、ablation study |
| **编号规则** | `EXP-XXX-简短描述.md`，从 001 递增 |
| **当前文档** | （暂无） |
| **何时阅读** | 复现实验、对比不同配置效果、追溯指标变化时 |
| **何时新建** | 每次有意义的训练/消融实验完成后 |

### 📅 journal/ — 每日工作日志

| 字段 | 说明 |
|------|------|
| **用途** | 记录每日进展、遇到的问题、待跟进事项 |
| **命名规则** | `YYYY-MM-DD.md`，按日期命名 |
| **当前文档** | （暂无） |
| **何时阅读** | 查看某天做了什么、当时遇到什么问题 |
| **何时新建** | 每个工作日结束时，或有重要进展/问题时 |

### 📦 archive/ — 归档文档

| 字段 | 说明 |
|------|------|
| **用途** | 已完结或被新文档取代的历史文档 |
| **命名规则** | 保持原编号和文件名不变 |
| **当前文档** | （暂无） |
| **何时阅读** | 需要查看历史分析或旧方案时 |
| **何时迁入** | 文档内容已被后续文档完全覆盖、或方案已废弃时 |

## 文档规范

1. 每个文档顶部应包含状态卡片（HTML 注释格式），包括 `id`、`title`、`type`、`status`、`created`、`updated`、`tags` 等字段
2. 分析文档的状态流转：`draft` → `active` → `done` | `superseded`
3. 文档间引用使用格式：`→ 参见 [ANAL-001]`
4. 被取代的文档在状态卡片中标注 `superseded_by`，并迁入 `archive/`
